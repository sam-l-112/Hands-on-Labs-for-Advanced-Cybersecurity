## 螢幕擷取範例 2：Canvas 畫壞（紅色）

**何時擷取**：點擊「立即更新 iOS 18.7」按鈕後，瀏覽器畫面中，Canvas 區域會瞬間變為全紅色並可能出現亂點或線條。
應立即擷取瀏覽器視窗（或僅 Canvas 區域）。

**應顯示內容**：
- Canvas 元素（`<canvas id="glcanvas">`）背景為鮮紅色（#FF0000 或 rgba(255,0,0,1)）
- 可能伴隨隨機像素點或線條（因為 `gl.clear` 只填充顏色，若後續有其他繪製則可能出現圖形）
- Canvas 邊框仍為紅色（由 CSS `border:2px solid red;` 決定）
- 建議擷取整個瀏覽器視窗以顯示按鈕與標題作為參考

**說明**：此圖證明學生成功觀察到 GPU 記憶體損毀的視覺化效果——WebGL 緩衝區被錯誤寫入導致顯示異常，是 exploit 第二步的關鍵證據。

**擷取提示**：
- 使用瀏覽器的「擷取螢幕畫面」功能（右鍵 → 擷取螢幕畫面）或作業系統截圖工具。
- 若想只擷取 Canvas，可右鍵點擊 Canvas → 「擷取節點」或使用開發者工具選取元素後擷取。
- 建議儲存為 PNG 格式，檔名建議：`screenshot_canvas_red.png`
